# Design System Repository

This repository contains a reusable design system with a theme toggle component library that can be published to npm.

## Repository Structure

This is a monorepo containing:

- **`apps/design-system/`** - Next.js application showcasing the design system
- **`src/`** - Reusable library code (theme toggle component)
- **`dist/`** - Built library files (generated by `npm run build:lib`)

## Quick Start

### Running the Design System App

```bash
npm install
npm run dev
```

Visit `http://localhost:3000/design-system` to see the component showcase.

### Building the Library

```bash
npm run build:lib
```

This generates the library files in `dist/`:
- `dist/index.js` (ESM)
- `dist/index.cjs` (CommonJS)
- `dist/index.d.ts` (TypeScript declarations)

## Publishing the Library

The theme toggle component can be published as an npm package: `@shru/theme-toggle`

**üìñ For complete library usage documentation, see [LIBRARY.md](./LIBRARY.md)**

### Quick Start

```bash
npm install @shru/theme-toggle
```

```tsx
import { ThemeToggle } from "@shru/theme-toggle"

function App() {
  return <ThemeToggle />
}
```

**CSS variables are automatically generated and loaded** - no manual CSS imports needed!

### Publishing

```bash
npm run build:lib
npm publish --access public
```

## Development

### Understanding the Structure

This design system follows a **module-based architecture**:

1. **Design System Modules** (`apps/design-system/src/design-system/`):
   - `components/` - UI component library
   - `themes/` - Theming system
   - Each module follows: `ui/`, `use[Module].tsx`, `[module]Config.js`, `[module]Utils.js`

2. **App-Level Modules** (root level):
   - `config/` - App configuration (site settings, fonts)
   - `hooks/` - App-level React hooks
   - `lib/` - Shared utilities (used by app and design-system)

3. **Library Code** (`src/`):
   - `components/` - Reusable theme toggle component
   - `tokens/` - Design tokens for runtime loading
   - `index.ts` - Library entry point

4. **Routes** (`apps/design-system/app/`):
   - Minimal route files that only import from design-system
   - All UI code lives in `src/design-system/components/ui/`

### Key Principles

- ‚úÖ **Routes are minimal** - Only import and render components
- ‚úÖ **UI in design-system** - All UI code belongs in `apps/design-system/src/design-system`
- ‚úÖ **Module structure** - Each module has: `ui/`, hooks, config, utils
- ‚úÖ **Centralized exports** - Use `index.ts` files for clean imports
- ‚úÖ **Library code separate** - Reusable code in `src/` for npm publishing
- ‚ùå **No business logic in routes** - Logic belongs in hooks or modules
- ‚ùå **No UI code in routes** - UI belongs in design-system
